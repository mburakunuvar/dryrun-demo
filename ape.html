Agentic Platform Engineering

Git-Ape — multi-agent Azure deployments, powered by GitHub Copilot

Git-Ape is a platform engineering framework built on GitHub Copilot. It provides a structured, multi-agent system for planning, validating, and deploying Azure infrastructure — with security gates, cost analysis, and CI/CD pipeline integration built in.

View on GitHub Get Started

01What it does

Git-Ape packages a set of Copilot agents and skills focused on Azure infrastructure work.

Requirements

Gather deployment requirements through conversational prompts.

Artifacts

Generate ARM templates and supporting deployment artifacts.

Pre-flight

Run security, preflight, and cost checks before deployment.

Lifecycle

Support onboarding and post-deployment validation.

Drift

Detect configuration drift between Azure and stored deployment state.

02The deployment flow

Seven stages, with blocking gates and explicit user confirmation.

1
Collect inputs
Requirements Gatherer interviews the user, validates CAF naming and SKU availability.
2
Generate & analyze
Template Generator produces ARM template, architecture diagram, cost estimate, and security report.
3
Security gate
Blocking check; failures loop back for fixes.
4
WAF review
Principal Architect scores against the 5 Well-Architected Framework pillars.
5
User confirmation
Nothing deploys without explicit approval.
6
Deploy
Resource Deployer runs az deployment, monitors, and retries.
7
Validate
Health checks, endpoint tests, architecture diagram generation.

03Agents

A coordinated set of specialists, orchestrated by a main agent.

@git-ape

Main orchestrator — coordinates stages, enforces security gates, delegates to subagents.

Requirements Gatherer

Interviews user, CAF naming, SKU validation.

Template Generator

ARM template, architecture diagram, cost estimate.

Resource Deployer

az deployment, monitoring, retry, integration tests.

Principal Architect

WAF 5-pillar review and trade-off analysis.

IaC Exporter

Import live resources into templates.

Git-Ape Onboarding

OIDC + RBAC, GitHub environments & secrets setup.

05Execution modes

Two ways to run Git-Ape — interactive in the IDE, or headless via the Coding Agent.

Interactive

VS Code Copilot Chat

Talk to @git-ape in VS Code Copilot Chat, authenticate via az login, and approve each step in real time.

Headless

Copilot Coding Agent

Coding Agent picks up a GitHub issue, generates templates on a branch, opens a PR, and CI/CD workflows handle validation, deployment, and teardown via OIDC.

06CI/CD workflows

GitHub Actions workflows that wire Git-Ape into your repository.

WorkflowTriggerAction
git-ape-plan.ymlPR with template changesValidate, what-if, post plan as PR comment
git-ape-deploy.ymlMerge to main or /deploy commentExecute ARM deployment
git-ape-destroy.ymlMerge PR with destroy-requestedDelete resource group
git-ape-verify.ymlManual dispatchVerify OIDC, RBAC, pipeline health
